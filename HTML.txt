+ 웹(HTML)이 중요한 이유
	+ 접근성, 개방성
	+ 예쁘게 만들기 위해 문자까지 이미지로 만들면 시각장애인들 같은 경우에는 해당 데이터를 알수가 없음
	
+ 태그를 각 기능 및 의미에 맞게 사용해야 웹페이지가 더 가치있어진다.

+ <h1>~<h6>
	+ 검색엔진이 제목으로 지정된 태그를 기준으로 우선적으로 결과를 보여준다.
	+ 단순히 제목처럼 보이게 만드는것보다 이런식의 디테일까지 챙기는게 중요하다.
	+ 비즈니스적인 측면에서도 중요해진다.

+ 리스트를 표현하는 부모태그와 자식태그
	+ <ul>
		+ Unordered List의 줄임말.
		+ <li>의 부모태그.
		+ 자식으로 갖는 <li>태그를 비슷한 분류끼리 그룹핑하는 용도로 사용한다.
	+ <ol>
		+ Ordered List의 줄임말.
		+ <li>의 부모태그.
		+ 자식으로 갖는 <li>태그를 자동으로 넘버링 해준다.(번호를 부여줌)

+ <title>
	+ 책으로 치면 표지에 해당한다.
	+ 검색엔진은 해당태그를 찾는다.
	  따라서 사용하지 않으면 엄청난 손해임!

+ 인터넷
	+ 초기에는 통신 시스템이었음
	
+ host : 인터넷에 접속되어있는 컴퓨터 각각을 지칭한다.

+ 웹 서버와 HTTP
	+ http://127.0.0.1/index.hml 에서 127.0.0.1은 IP주소(Internet Protocol Address)이다.
	  IP주소는 웹 브라우저가 설치된 각자의 컴퓨터의 웹서버를 가리킨다.
	  그중에서도 127.0.0.1는 모든 컴퓨터가 자기 자신을 가리키는 고정된 주소이다.
	+ 서로 다른 컴퓨터에 있는 웹 브라우저와 웹 서버가 통신을 하기 위해서는 반드시 http(Hyper Text Transfer Protocol)를 사용해야 한다.
